# ブレスト第2弾：キャラ育成 × 日常接触

## コアコンセプト
> キャラが勝手に自分の生活に入り込んでくる。能動的にアプリを開くんじゃなくて、日常のあちこちに「あ、またいる」が発生する。

## キーインサイト（ブレストで出た重要な気づき）

- **たまごっち的な「お世話・愛着・自分だけの子」が核**
- ポケモン（コレクション・バトル）よりたまごっち（お世話・愛着・個別性）寄り
- **いかにユーザーとキャラの接触回数を"無意識に"増やすか**が勝負
- 「大人が幼いものを楽しむ」文化は既に成立している（キダルト市場：世界のおもちゃ売上の28%が大人）
- 成功IPの共通点：「かわいい × 予想外の何か」のギャップが必要
- キティの口がないデザイン → キャラに「余白」があると長寿IPになる

---

## 案1: キーホルダーの中にキャラが住んでる

### コンセプト
物理的なキーホルダーの中にキャラが「住んでいる」。スマホをかざす（読み取る）とキャラが出てきて、お世話できる。他の人のキーホルダーを読み取ると、許可制でキャラを同じ部屋に一緒にいさせられる。

### ポイント
- **物理的な存在感**がある（アプリは見えないけどキーホルダーは見える）
- 学校やバイト先で「なにそれ？」が自然に発生する
- たまごっちが持ってた「物理的に持ち歩く」体験をそのまま再現
- 友達のキーホルダーを読み取って会わせる行為自体が遊びになる
- キーホルダー自体の物欲でも広がる（「あの子のかわいい」）

### 未決事項
- 読み取り技術（NFC / QRコード / ARマーカー / Bluetooth）
- キーホルダーの形態（アクリル / 画面付き / 光る / アナログ見た目）
- キーホルダーの販売・流通方法

---

## 案2: キャラが自分の代わりにSNSやってくれる

### コンセプト
自分のキャラがSNSを代わりにやってくれる。ユーザーの写真フォルダから勝手に選んで、キャラの視点で投稿する。自分は発信しなくていいけど、キャラが勝手に日常を発信してくれる。

### ポイント
- **SNS疲れ世代への逆張り**：自分で発信するプレッシャーゼロ
- キャラが写真を選ぶ → 「え、それ選ぶ？ｗ」が面白い
- キャラの視点でのコメントが個性的（ユーザーが書くより面白くなる可能性）
- 友達がキャラの投稿を見る → 間接的にユーザーの日常を知る
- **写真フォルダから勝手に選ぶ**というのがキャラの自律性を感じさせる

### 未決事項
- プラットフォーム（専用SNS / 既存SNS連携 / 両方）
- 写真選択のロジック（AI判定 / ランダム / ユーザー事前承認）
- プライバシー設計（見せたくない写真の除外方法）

---

## 案4: キャラが写真に勝手に写り込む

### コンセプト
カメラを起動するとキャラがAR的に写真に入り込んでくる。自撮りにも風景にも。キャラの気分によって写り込み方が変わる（機嫌いいと前に出てくる、悪いと隅っこにいる）。写真を撮るたびにキャラとの思い出が増える。

### ポイント
- **写真を撮るという日常行動**にキャラが乗っかる → 接触回数が自然に増える
- キャラの気分で位置が変わる → 毎回違って飽きない
- 写真にキャラがいる = シェアした時に自然にキャラが広まる
- 「今日うちの子めっちゃ端っこにいるんだけどｗ」がネタになる
- 過去の写真を見返すとキャラとの思い出が蓄積されてる

### 未決事項
- AR技術の精度・コスト
- 既存カメラアプリとの連携 or 専用カメラ
- キャラの気分を決めるロジック

---

## 案5: キャラ同士が勝手に友達になる

### コンセプト
自分のキャラと友達のキャラが、近くにいると（Bluetooth等）勝手に出会う。キャラ同士が仲良くなったり、喧嘩したり、付き合い始めたりする。自分たちの関係ではなく、キャラ同士の関係が自律的に進む。

### ポイント
- **「え、うちの子とあんたの子付き合い始めたんだけどｗ」**が最強のバイラルネタ
- 自分の意思でコントロールできない → 予想外が起きる → 話のネタになる
- 友達と会うだけで勝手にコンテンツが生まれる
- キャラの関係図が複雑になっていく → 長期で飽きない
- 少人数（2人）でも成立する

### 未決事項
- 近接検知の技術（Bluetooth / 位置情報 / 手動）
- キャラ同士の関係性ロジック（ランダム / 性格ベース）
- 関係性の通知・表示方法

---

## 4案の共通構造

全て「**キャラが勝手に動く → ユーザーの日常に無意識に接触**」という構造。

| 案 | 接触ポイント | キャラの自律性 |
|---|---|---|
| 案1 | カバンにつけてるキーホルダー | キーホルダーの中で生きてる |
| 案2 | SNSのタイムライン | 勝手に写真選んで投稿する |
| 案4 | カメラ起動時 | 勝手に写真に入り込んでくる |
| 案5 | 友達と会った時 | 勝手に友達のキャラと関係を築く |

### 補足：1つのサービスの中で複数成立する可能性あり
この4つはバラバラのサービスではなく、「キャラが生活に住みつく」という1つのコンセプトの中の複数の接触ポイントとして統合できる可能性がある。
