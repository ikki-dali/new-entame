# 案C: 位置情報×キャラSNS「まちキャラ」

## 一言で言うと
> 自分だけのキャラが実在の街に"住んでいる"。カフェに行くとキャラがそこにいて反応する。友達のキャラと自分のキャラが勝手に交流して、その日記が毎日届く。

## コンセプト

### 体験の流れ
1. 初回起動 → 自分のキャラを作成
2. キャラは自分の「よく行く場所」に住み始める
3. 実際にどこかに行くと、キャラがその場所に反応（テキスト+イラスト）
4. 友達のキャラと自分のキャラが、場所が近いと**勝手に出会って交流**
5. 毎日夜に「キャラの日記」が届く：「今日は渋谷でゆきのキャラと会ったよ！」
6. キャラは場所の雰囲気に影響されて性格や見た目が変化

### キャラの日記の例
```
📍 今日のまちキャラ日記

朝: おうちでゴロゴロしてた
昼: 渋谷のカフェに連れてってもらった！
    ラテアートかわいかった〜☕
    そしたら偶然ゆきちゃんのコに会った！
    一緒にウィンドウショッピングしたよ🛍
夜: 今日は楽しかった。
    最近おしゃれな場所ばっかりで
    ちょっとキラキラ系に近づいてきたかも？✨

📊 性格変化: おとなしい → ちょっと社交的に
```

## ターゲットに刺さる理由

### リアル生活との融合
- 「今日どこ行った」がキャラを通して可視化される
- 日常が"物語"になる感覚
- **Zenly的な「友達の位置を知る楽しさ」をキャラで間接的に表現**（プライバシー配慮）

### キャラ×場所の新鮮さ
- 原宿に住むキャラは原宿っ子に
- 図書館によく行くキャラは読書家に
- **自分の生活がキャラの個性に反映される**

## バイラル設計

### 自然拡散ポイント
1. **キャラの日記シェア**: 「今日うちの子が渋谷でこんなことしてたｗ」
2. **場所×キャラの発見**: 「ここ行ったらキャラがこう反応した！」
3. **キャラ同士の出会い**: 「うちの子とあなたの子が会ってたらしいよ！」
4. **聖地巡礼**: 特定の場所に行くとレアイベント

### ローカルバイラル
- 学校・バイト先でキャラが反応 → 周りの子も始める
- 地域密着型の拡散（同じエリアの人から広がる）

## 少人数での楽しみ方
- **1人**: キャラの日記を読む。自分の行動がキャラに反映される楽しさ
- **2-3人**: キャラ同士の交流が発生。一緒に出かけるとキャラも一緒
- **地域**: 同じエリアの知らない人のキャラと偶然の出会い

## IP展開の可能性
| 展開 | 内容 |
|------|------|
| ローカルIP | 渋谷のキャラ、原宿のキャラなど地域キャラ化 |
| 商業施設コラボ | 特定の店に行くとコラボキャラに会える |
| ご当地グッズ | 旅行先のキャラグッズ |
| ARイベント | リアルイベントでキャラがAR表示 |

## 技術要件（概算）
- **位置情報API**: GPS連携、ジオフェンシング
- **LLM API**: キャラの日記・反応生成
- **地図データ**: 場所の属性情報（カフェ、公園、学校等）
- **バックエンド**: 位置情報処理、キャラの状態管理
- **フロントエンド**: iOS/Androidアプリ
- **MVP開発期間**: 4-6ヶ月（位置情報周りが複雑）

## リスク
- **位置情報のプライバシー問題**（Zenly終了の理由でもある）
- 開発コストが高い（位置情報+AI+キャラ）
- バッテリー消費
- 屋内での精度問題
- 地方ユーザーは体験が薄くなる可能性
