# 案A: 1日1回AIキャラチャット「うちの子」

## 一言で言うと
> 毎日ランダムな時間に届く通知。あなた専用のAIキャラと「今日の1回だけ」の会話ができる。キャラはあなたの返答で性格が変わっていく。

## コンセプト

### 体験の流れ
1. アプリ初回起動 → キャラの見た目・初期性格を選ぶ（or ガチャ）
2. 毎日ランダムな時間に通知が届く：「○○（キャラ名）が話しかけてるよ！」
3. キャラとの会話は**1日1回、数ターンだけ**
4. あなたの返答によって、キャラの感情・性格・見た目が少しずつ変化
5. 会話の結果が「カード」として生成される → スクショ映えするデザイン
6. 友達と「うちの子」を見せ合う文化が生まれる

### なぜ1日1回なのか
- **希少性 = 価値**: 毎回の会話が特別になる
- **BeRealの法則**: 制約があるから「やらなきゃ」と思う
- **継続率UP**: 毎日少しだけ → 習慣化しやすい
- **開発コスト**: API呼び出し回数を制御できる

## ターゲットに刺さる理由

### 「うちの子」文化
- 10代〜20代前半女性は「推し」「うちの子」文化が根強い
- 自分だけのキャラ = 所有欲 + 愛着
- キャラが"成長する"感覚 → たまごっち的中毒性
- **「うちの子がこんなこと言った！」がSNSでバズるコンテンツになる**

### SNS疲れへの処方箋
- 自分が発信しなくていい（キャラが話す）
- 比較・マウントが生まれにくい
- 1対1の親密な空間

## バイラル設計

### 自然拡散ポイント
1. **スクショ文化**: キャラの面白い/かわいい発言をスクショ → TikTok/X/Instagramストーリーに投稿
2. **「うちの子紹介」**: 友達同士で「うちの子見て！」のシェア
3. **キャラの変化**: 「3ヶ月育てたらこうなった」のビフォーアフター
4. **レアイベント**: 低確率で起きる特殊反応 → 「これ出た！」で拡散

### TikTok攻略
- 「AIキャラに〇〇って言ったらこうなったｗ」系動画
- 育成結果のビフォーアフター
- キャラの名言集

## 少人数での楽しみ方
- **1人**: メインの体験。1人で完全に成立する
- **2-3人**: 「うちの子」を見せ合う。キャラ同士を会わせる機能
- **グループ**: キャラの人気投票、キャラ同士のイベント

## IP展開の可能性
| 展開 | 内容 |
|------|------|
| LINEスタンプ | 自分のキャラのスタンプを生成 |
| グッズ | アクスタ、ぬいぐるみ（自分のキャラを注文） |
| コラボ | 実在ブランドとキャラのコラボ衣装 |
| メディア | 人気キャラの漫画化・アニメ化 |

## 技術要件（概算）
- **LLM API**: キャラの会話エンジン（GPT-4o / Claude等）
- **キャラデザ生成**: イラスト生成AI or デザイナー制作のベーステンプレート
- **バックエンド**: キャラの状態管理、ユーザーデータ
- **フロントエンド**: iOS/Androidアプリ（React Native or Flutter）
- **MVP開発期間**: 2-3ヶ月（最小構成）

## リスク
- LLM APIのコスト（1日1回に制限することで緩和）
- キャラの発言品質管理（不適切発言の防止）
- Character.AIなど既存サービスとの差別化
- キャラデザのクオリティがバイラルに直結

## 差別化ポイント（vs Character.AI）
| | Character.AI | うちの子 |
|---|---|---|
| 会話頻度 | 無制限 | 1日1回（希少性） |
| キャラ | 既存キャラ中心 | 自分だけのオリジナル |
| 成長 | なし | 会話で性格・見た目が変化 |
| シェア | 会話ログ | 映えるカードデザイン |
| IP展開 | なし | グッズ、スタンプ等 |
